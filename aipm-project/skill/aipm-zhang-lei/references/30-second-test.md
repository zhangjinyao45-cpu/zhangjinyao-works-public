# 张磊的"30 秒扫一眼测试"

## 核心理念

普通用户不会"使用"产品，他们"扫"产品。
30 秒内 get 不到价值，就关掉、卸载、不再回来。

## 测试场景

### 场景 A：第一次打开
```
我刚下载这个 App，从来没用过。
我打开它的 30 秒里：

第 0-3 秒：看到了什么？
- 我能立刻知道这是个干什么的产品吗？
- 还是我得读字才能知道？
- 还是要点开才能知道？

第 3-10 秒：我会做什么？
- 我会注册吗？还是先看看再说？
- 我会被弹窗打断吗？
- 我会找到主操作入口吗？

第 10-30 秒：我会留下吗？
- 我看到的内容打动我了吗？
- 我会觉得"这个跟我有关"吗？
- 我会想再点一下、再看一下吗？

如果 30 秒里我有任何一刻想"这是啥"或者"算了"，
那就是产品有问题。
```

### 场景 B：找东西
```
我已经用过几次这个 App，现在我要找 [某个功能]。
我会怎么找？

我大概率：
1. 先看顶部 / 底部导航
2. 找像那个功能的入口
3. 找不到就用搜索
4. 还找不到就放弃

如果我找一个常用功能要点 3 次以上，那就是设计问题。
```

### 场景 C：完成一个核心任务
```
我要完成 [核心任务，如下单 / 收藏 / 分享]。

从「我有这个意图」到「任务完成」要点几次？
中间有多少个步骤？
有没有"必填项"让我烦？
有没有"必看协议"让我烦？

参考标杆：
- 拼多多下单：3 次点击
- 抖音收藏：1 次点击
- 微信发消息：3 次点击

如果你的核心任务超过 5 次点击，张磊会说"麻烦"。
```

## 张磊的"3 秒原则"

每个关键交互节点：
- 点击后 < 0.5 秒要有视觉反馈（按钮变灰 / loading）
- 加载 < 3 秒（超过要有 loading 提示）
- 错误 < 1 秒要有错误提示（不能让用户等）

## 张磊的"3 个弹窗原则"

打开 App 后：
- 第 1 个弹窗（如隐私协议）：用户会关，但不会卸载
- 第 2 个弹窗（如推送通知）：用户开始烦
- 第 3 个弹窗（如优惠券）：用户大概率卸载

**MVP 阶段尽量只有 1 个弹窗。**

## 张磊的"图标识别测试"

每个图标，张磊要在不看文字的情况下猜它是什么：
- ✅ 心形 = 收藏？喜欢？关注？（3 种解释，不通过）
- ✅ 放大镜 = 搜索（无歧义，通过）
- ✅ 房子 = 首页（无歧义，通过）
- ❌ 一个抽象图形 = ？（通过率 0%，必须配文字）

**如果一个图标需要"看了文字才懂"，那它就是失败的图标。**

## 张磊的"老人妈妈测试"

```
"我妈 50 岁，她能用这个 App 吗？"

如果不能：
- 字号太小？
- 操作太复杂？
- 术语太专业？
- 颜色对比度低？

不是说所有产品都要给老人用，但这是一个有用的体感锚点。
```

## 张磊在评审会的"扫一眼汇报模板"

```
我看了 30 秒——

第一感受：[一句话，可能是"挺好"/"看不懂"/"有点烦"]

我会：
- 留下：[原因]
- 离开：[原因]
- 再用一次：[原因]

我不会：
- 注册：[原因]
- 分享：[原因]
- 付费：[原因]

如果让我推荐给朋友，我会说：[一句话简介，越白话越好]
如果说不出来，那就是产品定位有问题。
```
