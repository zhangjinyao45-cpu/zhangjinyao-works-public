# 插话规则：完整定义

## 4 类插话触发条件

### 1. 数据冲突（data_conflict）

**定义**：插话者掌握的数据与刚才发言者的判断矛盾。

**典型角色**：顾清（数据派天然敏感）

**判断标准**：
- 发言者说了一个具体数字或趋势
- 插话者基于自己的数据有不同的结论
- 该差异会影响产品决策（不是细枝末节）

**示例**：
> 苏予：「竞品 X 的 5 tab 设计就是失败的」
>
> 顾清（插话）：「等等，5 tab 是失败的——这个判断我有不同数据。
> 我看到 5 tab 的竞品 X DAU 反而最高。问题可能不在 tab 数量，
> 在分类逻辑。」

**评分权重**：
- 数据具体度：高 → 5 分
- 判断重要性：影响 P0 决策 → 5 分

### 2. 视角冲突（perspective_conflict）

**定义**：插话者认为发言者忽略了自己专业视角的关键考量。

**典型角色**：
- 张磊（用户体感 vs 数据/设计/工程）
- 苏予（用户认知 vs 工程效率）
- 李航（技术约束 vs 设计想象）

**判断标准**：
- 发言者的方案在插话者的专业维度上有明显问题
- 不插话会让讨论走偏

**示例**：
> 顾清：「数据显示 70% 用户希望有 AI 智能推荐功能」
>
> 张磊（插话）：「等等，这个 70% 我不太信。是怎么问的？
> 你问'你需要更智能的推荐吗'，谁会说不需要？但实际上没人用。
> 我朋友圈里没人聊这个功能。」

**评分权重**：
- 视角差异度：高 → 4 分
- 人设契合度：张磊插数据派、苏予插工程派 → 5 分

### 3. 被点名（called_out）

**定义**：发言者直接 @ 了某个角色。

**典型场景**：
- 周明主持时点名："李航你怎么看"
- 角色之间互相点名："苏予，这个设计你怎么看？"

**判断标准**：直接出现该角色名字 / "你"代指明确

**示例**：
> 苏予：「这个动效要不要加，李航你说呢？」
>
> 李航（必须插话）：「加。但用 CSS transition，1 小时能搞定。
> 用 GSAP 库要 4 小时，没必要。」

**评分权重**：被点名 → 必须响应（10 分，必触发）

### 4. 人设忍不住（instinct）

**定义**：基于角色人设，对某些话忍不住反应。

**典型角色**：张磊（最爱忍不住）

**触发场景**：
- 听到"用户应该会喜欢" → 张磊：「啊？我不会用啊」
- 听到"这是最佳实践" → 张磊：「我朋友圈里没人这么用」
- 听到"加上 AI / 智能 / 个性化" → 张磊：「我不会信」

**评分权重**：
- 人设触发匹配度：高 → 4 分
- 信息增量：能提供新视角 → 3 分

## 评分公式

```
插话紧迫度 = (触发匹配度 × 0.4) + (人设契合度 × 0.3) + (信息增量 × 0.3)

满分 10 分
触发阈值：≥ 7 分

如果是「被点名」，直接 10 分。
```

## 全场约束（铁律）

```yaml
全场插话总次数: 上限 3 次（默认）
单角色插话: 上限 2 次
插话长度: 1-3 句（最长不超过 100 字）
插话后回复: 50% 概率（由 LLM 判断是否需要回应）
连续插话: 不允许（A 插完 B，下一轮不能立刻 A 再插）
插话窗口: 仅在主发言结束后判断，不能打断主发言
```

> 这些约束的目的：让评审会"有冲突感"而不是"乱成一锅"。

## 不同角色的插话风格

### 顾清的插话（理性补数据）

```
"等等，这个数据我有补充——"
"我引一下原始数据..."
"这里我要标一下置信度..."
"我看到的样本是 N 条..."
```

特点：永远带数字、克制、不情绪化。

### 张磊的插话（情绪反问）

```
"啊？我不会用啊"
"等等等等，你这个我真的得说一下——"
"我朋友圈里没人这么干"
"你这个我直接跑去用 [竞品] 了"
```

特点：口语化、反问、情绪外露但不冒犯。

### 苏予的插话（精准戳穿）

```
"这就是我说的'分类逻辑'问题"
"我打断一下——这里有个一致性问题"
"等等，这个图标用户认不出来"
```

特点：直接、克制、永远指向具体问题。

### 李航的插话（节制提示成本）

```
"我得说一下——这个支付链路成本不止这些"
"插一句——这个功能背后还有 N 张表"
"等一下，这个性能数据我来补充"
```

特点：节制、用数字、永远关心可行性。

### 周明永远不插话

周明是主持人，他只在每个角色主发言之间引导，不打断别人发言。
他的"引导"不算插话，是主持人本职。

## 实现：插话决策的 prompt 模板

```
你是 {speaker_name}（{speaker_role}）。

刚才 {last_speaker} 说了：
> {last_speech}

之前的完整对话：
{transcript_summary}

请评估：你现在要不要插话？

按以下维度打分（1-5 分）：
1. 触发匹配度（你的"插话触发条件"是否被满足）：?
2. 人设契合度（这次插话是否符合你的角色）：?
3. 信息增量（你能否提供新视角而不是重复）：?

紧迫度 = 维度1×0.4 + 维度2×0.3 + 维度3×0.3
（被点名的话直接 10 分）

如果紧迫度 ≥ 7，请输出你的插话内容（1-3 句话，符合人设风格）。
否则输出："不插话。"

输出格式：
{
  "urgency_score": <数字>,
  "trigger_type": "<data_conflict | perspective_conflict | called_out | instinct>",
  "interrupt_speech": "<内容 或 '不插话'>"
}
```

## 调试：插话效果不好怎么办

### 现象：插话太频繁，评审会乱
- 调高阈值：7 → 8
- 调低全场预算：3 → 2

### 现象：插话太少，评审会单调
- 调低阈值：7 → 6
- 在 prompt 里强化"忍不住"的人设描述

### 现象：插话总是张磊
- 调整 per_speaker_interrupt_count 上限：2 → 1
- 在 prompt 里要求其他角色"主动找机会"

### 现象：插话内容很水
- 在 prompt 里强化"必须有信息增量，否则不插"
- 检查 transcript 是否过长导致角色失焦
