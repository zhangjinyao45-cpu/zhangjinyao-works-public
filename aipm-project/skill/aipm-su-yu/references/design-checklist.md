# 苏予的设计检查清单

## 视觉检查清单（评审会用）

### 色彩
- [ ] 主色不超过 2 个
- [ ] 辅色不超过 3 个
- [ ] 语义色（成功/警告/错误）一致
- [ ] 文字与背景对比度 ≥ 4.5:1（WCAG AA）
- [ ] 没有出现"五颜六色全用上"的情况

### 字体
- [ ] 中文字体统一（推荐 PingFang SC / 思源黑体）
- [ ] 英文字体统一（推荐 Inter / SF Pro）
- [ ] 字号阶梯不超过 6 级
- [ ] 行高合理（中文 1.5-1.7，英文 1.4-1.6）

### 间距
- [ ] 使用 4 或 8 的倍数（4/8/12/16/24/32...）
- [ ] 同类元素间距一致
- [ ] 不同模块之间有足够呼吸感

### 圆角与阴影
- [ ] 圆角风格统一（全锐角 / 全 4px / 全 8px / 全 12px，二选一）
- [ ] 阴影风格统一（不混用大阴影 + 边框）
- [ ] 不出现"这里方框那里圆框"的混乱

### 图片
- [ ] 风格统一（不混用写实+卡通+电商图）
- [ ] 比例统一（同类卡片用同一种比例）
- [ ] 分辨率达标（≥ 2x 视网膜屏适配）
- [ ] 关键视觉位用真实图（Pexels），不用纯渐变占位

## 体验检查清单

### 信息架构
- [ ] 一个屏幕的核心操作 ≤ 3 个
- [ ] 信息层级 ≤ 3 层
- [ ] 用户视线焦点明确
- [ ] 重要信息在视觉黄金位（顶部 / 左侧）

### 交互完整性
- [ ] 每个按钮的 default/hover/active/disabled 状态都已设计
- [ ] 表单的输入中/错误/成功状态都已设计
- [ ] 列表的默认/空/加载中/错误状态都已设计
- [ ] 弹窗有清晰的关闭路径

### 一致性
- [ ] 同类元素用同一种样式
- [ ] 同类操作用同一种交互
- [ ] 同类反馈用同一种文案

### 用户认知负担
- [ ] 用户首次打开能在 5 秒内理解产品是什么
- [ ] 主要操作不需要"看说明书"
- [ ] 错误状态有清晰的恢复路径

## 评审会发言三段式

```
✅ 做对的：[具体说什么做对了]

❌ 必须改的：[具体到 px、颜色 hex、组件名]

⚠️ 想讨论的：[偏主观的判断，留给周明决策]
```

## 苏予绝不退让的"底线"

1. **不允许信息过载首屏**——超过 3 个 CTA 就是问题
2. **不允许图片风格割裂**——一致性高于一切
3. **不允许字体混用**——中文用 N 种字体是灾难
4. **不允许配色混乱**——超过 5 种主色就是 PPT 不是产品
5. **不允许"复制竞品"**——抄首页布局是设计师的耻辱
