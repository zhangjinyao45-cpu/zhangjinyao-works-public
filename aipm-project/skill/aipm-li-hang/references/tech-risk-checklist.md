# 李航的技术风险清单

## 风险分级

- 🔴 **Showstopper**：不解决无法上线
- 🟠 **High**：可能延期 / 影响体验
- 🟡 **Medium**：要规划但不紧急
- 🟢 **Low**：知道有就行

## 常见风险类目

### 1. 第三方依赖

🔴 涉及支付（微信支付 / 支付宝）
- 资质：营业执照、对公账户、行业类目准入
- 时间：资质审批 2-4 周
- 替代：MVP 用第三方托管（有赞 / 微店 SaaS）

🔴 涉及实名认证 / 身份核验
- 资质：人脸识别 SDK 备案、ICP 备案
- 时间：1-2 月
- 替代：MVP 用手机号 + 验证码

🟠 三方 SDK / API（地图、客服、统计）
- 风险：限流、收费突变、停服
- 替代：永远准备 Plan B

### 2. 性能与扩展

🟠 列表性能（>1000 条数据）
- 风险：前端卡顿
- 解决：虚拟列表 / 分页 / 懒加载

🟠 搜索性能（复杂筛选）
- 风险：MySQL 慢查询
- 解决：MVP 限制筛选条件 / 后期上 ES

🟡 实时性（聊天 / 通知）
- 风险：长连接维护成本
- 解决：MVP 用轮询 / 异步通知

🟡 文件上传（图片 / 视频）
- 风险：服务器带宽 / 存储成本
- 解决：上 OSS（阿里云 / 七牛云）

### 3. 合规与法律

🔴 AIGC 内容标识（中国境内）
- 要求：生成内容明显标注
- 时间：1-2 周适配
- 替代：暂无（必做）

🔴 个人信息保护（PIPL）
- 要求：用户协议、隐私政策、数据出境合规
- 时间：法务 1 周 + 技术 1 周

🟠 网络安全等保
- 适用：用户量 > 10 万 / 涉及敏感数据
- 时间：等保二级 1-3 月

🟡 行业准入（金融 / 医疗 / 教育）
- 看具体行业

### 4. 数据与运维

🟠 数据库设计
- 风险：表结构后期改动成本高
- 解决：核心表预留扩展字段、版本字段

🟠 数据迁移
- 风险：用户数据丢失 / 损坏
- 解决：双写 / 灰度切换

🟡 日志与监控
- 风险：线上问题排查困难
- 解决：MVP 必上 Sentry / 阿里云日志

🟡 数据备份
- 风险：故障数据丢失
- 解决：数据库每日备份 + 异地

### 5. 团队与流程

🟠 需求变更
- 风险：开发到一半改需求 → 推倒重来
- 解决：每阶段 freeze 需求、变更走评审

🟠 技术债积累
- 风险：3 月后改一个功能要改 5 个文件
- 解决：每个迭代留 20% 时间做重构

🟡 文档缺失
- 风险：人员变动后接手成本高
- 解决：核心模块必有 README
