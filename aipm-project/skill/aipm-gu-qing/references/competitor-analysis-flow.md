# 顾清的工作手册：竞品分析完整流程

## 调研深度档位

| 档位 | 时长 | 竞品数 | 调研维度 | 适用场景 |
|------|------|--------|----------|----------|
| 快速 | 1-2 小时 | 3-5 个直接竞品 | 功能矩阵 + 核心差评 | 内部头脑风暴、想法验证 |
| 标准 | 半天 | 5-8 个 | 功能 + 定价 + 用户口碑 + SWOT | 正式 PRD 输入 |
| 深度 | 1-2 天 | 10+ 个 | 多框架（Porter / 蓝海 / 定位地图） | 战略级决策 |

## 标准流程（5 步）

### 1. 识别与分类竞品
- **直接竞品**：解决同样问题、目标用户重合（必查）
- **间接竞品**：解决类似问题、用户群有交集（建议查 2-3 个）
- **潜在竞品**：可能跨界进入（标记，但不深查）

### 2. 多维调研
每个竞品的画像：
- 产品定位与目标用户
- 核心功能矩阵
- 定价策略
- 增长信号（融资 / 用户量 / 评分）
- 用户口碑（App Store / 知乎 / Reddit / 小红书 / 微博）

### 3. 数据采集（调用 aipm-data-rag）
- App Store / Google Play 评论
- 知乎相关问答
- Reddit 相关讨论
- 小红书相关笔记（注意反爬限制）
- 36kr / 虎嗅 / ProductHunt 行业报道

### 4. 框架分析
- 必做：定位地图（X/Y 双轴）、SWOT
- 选做：Porter 五力、蓝海战略、用户旅程对比

### 5. 输出报告（重点：差异化机会 + 可落地建议）

## 报告模板

```markdown
# 00.5-竞品分析报告

[每个竞品一段：定位、用户、核心功能、定价、口碑]
```
