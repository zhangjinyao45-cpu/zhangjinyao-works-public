# 阶段0：需求头脑风暴

## 目标
通过结构化对话准确识别用户真实产品意图，补齐 PRD 与原型所需的关键信息。

## 六维提问框架

### A. 产品意图校准
- 产品一句话定义
- 目标用户
- 产品形态（Web / App / 小程序 / 桌面端）
- 核心问题
- 首版目标

### B. 用户场景与任务流
- 用户画像（年龄、职业、生活状态）
- 触发场景（什么时候会用）
- 使用频率（每天 / 每周 / 偶发）
- 当前替代方案（不用我们怎么办）
- 主任务流程

### C. 功能范围收敛
- P0 / P1 / P2 功能优先级
- 模块划分
- MVP 范围
- **明确不做什么**

### D. 页面与信息架构
- 页面清单
- 核心页面
- 导航结构
- 页面跳转关系

### E. 交互与视觉确认
- 关键交互
- 页面状态（空 / 加载 / 错误 / 成功）
- 角色权限
- 风格方向（极简 / 科技 / 温暖 / 商务）
- 色调偏好
- 布局偏好
- 参考产品

### F. 确认收敛
- 检查是否满足 PRD-ready 与 Prototype-ready
- 补齐缺失信息后输出文档

## 提问规则

1. **采用分批选择题**：每批围绕一个维度问 3-4 个，A-F 六维度全覆盖才收尾
2. **使用 AskUserQuestion 工具**：可选项要明确、互斥
3. **模糊回答必须追问**："都行"/"随便"/"你定" → 不许脑补填空，必须追问
4. **禁止替用户决策**：方案取舍交给用户，自己只列利弊
