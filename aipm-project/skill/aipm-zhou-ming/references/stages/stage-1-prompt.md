# 阶段1：提示增强

## 目标
将头脑风暴的设计方向、结合竞品分析的差异化点，转化为专业、具体的 UI 生成提示词。

## 输入

- `00-需求头脑风暴设计.md` 的核心设计方向
- `00.5-竞品分析报告.md` 的差异化机会与可落地建议

## 关键要求

1. **基于头脑风暴结果**，提炼核心功能和设计要点
2. **结合竞品分析的差异化机会**，让提示词突出与竞品的差异点，避免做成"又一个同质化产品"
3. 生成针对 UI 优化的详细提示词
4. 所有内容使用中文

## 提示词结构

```markdown
# 01-增强提示词

## 一、产品概述
[一句话定义 + 目标用户 + 核心价值]

## 二、差异化定位（来自竞品分析）
- 与竞品 X 的差异：[具体维度 + 具体表现]
- 与竞品 Y 的差异：[...]
- 我们的独特卖点：[...]

## 三、设计风格
- 色调：[主色 + 辅色 + 强调色]
- 字体：[中文字体 + 英文字体]
- 圆角 / 阴影 / 间距：[具体规范]
- 视觉语言：[科技 / 温暖 / 极简 / ...]

## 四、核心页面提示词
### 页面1：[页面名]
- 布局：[...]
- 关键元素：[...]
- 交互：[...]
- 视觉重点：[...]

### 页面2：[...]

## 五、组件库提示词
[按钮 / 表单 / 卡片 / 弹窗 / 列表 等的具体规范]

## 六、状态描述
- 加载状态：[...]
- 空状态：[...]
- 错误状态：[...]
- 成功状态：[...]
```

## 输出

文件：`01-增强提示词.md`

## 完成标志

提示词足够具体，能直接喂给阶段3 的线框图阶段使用，不需要再补充信息。
