# 阶段2：生成 PRD

## 目标
创建完整的产品需求文档，让研发、设计、运营拿到这份文档就能开干。

## 输入

- `01-增强提示词.md`
- `00.5-竞品分析报告.md`

## PRD 文档结构

```markdown
# 02-产品需求文档

## 一、产品定位
- 产品名称
- 一句话定义
- 目标用户（画像 + 场景）
- 核心问题与解决方案
- 产品愿景

## 二、市场与竞争（引用 00.5-竞品分析报告.md）
- 市场规模简评
- 直接竞品概况
- 我们的差异化定位
- 竞争策略

## 三、功能清单（含优先级）
| 模块 | 功能 | 优先级 | 说明 | 数据依据 |
|------|------|--------|------|----------|
| ...  | ...  | P0     | ...  | 顾清调研：xxx |

## 四、用户旅程
[关键用户流程，从触达到转化的全链路]

## 五、页面结构
- 页面清单
- 页面层级
- 页面跳转关系图

## 六、交互规则
- 关键交互定义
- 页面状态规则
- 错误处理规则

## 七、非功能需求
- 性能要求
- 兼容性要求
- 安全 / 合规
- 可访问性

## 八、风险与依赖
[李航在评审会上提出的技术风险 + 顾清的市场风险]

## 九、成功指标（北极星指标）
- 主指标
- 辅助指标
- 验证假设
```

## 关键要求

1. **每个功能优先级判断要有数据依据**——引用顾清竞品分析的结论
2. **差异化定位要具体**——不是"我们更好"，是"我们在 X 维度做到 Y，竞品做不到"
3. **风险要诚实**——李航在评审会提的所有技术风险都要写进去
4. **指标要可验证**——不写"提升用户体验"这种没法量化的目标

## 输出

文件：`02-产品需求文档.md`

## 评审会重点

PRD 阶段的评审会，4 位角色重点挑战：
- 顾清：差异化定位的市场依据是否充分？
- 张磊：从用户视角看，这个 PRD 解决我的问题吗？
- 苏予：这些功能加起来是不是太复杂？信息架构合理吗？
- 李航：这个 PRD 的工时大概多少？哪些功能技术风险高？
