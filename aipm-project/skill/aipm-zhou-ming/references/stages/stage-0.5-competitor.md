# 阶段0.5：竞品分析（协同顾清）

## 目标
基于阶段0 的产品定义，对标赛道竞品，输出可直接支撑后续设计的竞争格局与差异化结论。

## 周明在这个阶段的职责

**这个阶段顾清主导**（调用 `aipm-gu-qing` skill），周明负责：

1. **从阶段0 产物中提取关键输入**
   - 产品定义、目标用户、核心问题
   - 用户已经提到过的竞品（如"类似 XX"）
   - 用户的差异化期望

2. **与用户确认调研深度**
   - 快速档（1-2 小时）：3-5 个直接竞品，功能矩阵 + 核心差异
   - 标准档（半天）：5-8 个竞品，含定价 / 用户口碑 / SWOT
   - 深度档（1-2 天）：10+ 竞品，多框架分析（Porter / 蓝海 / 定位地图）

3. **把关顾清的产出**
   - 「差异化机会」是否明确、可执行？
   - 「可落地建议」是否具体到能写进 PRD？
   - 数据来源是否真实、可追溯？
   - 是否标注了置信度和数据缺口？

4. **触发评审会**：让张磊、苏予、李航也对竞品分析结论挑战

## 输入

- `00-需求头脑风暴设计.md`（产品定义、目标用户、已知竞品）

## 输出

- `00.5-竞品分析报告.md`（顾清产出）
- `00.5-评审会纪要.md`（周明主持的评审记录）
- 决策点清单：选择哪条差异化路径

## 与顾清的协作

```
周明：从阶段0 提取产品定义 + 已知竞品
  ↓
周明：调用 aipm-gu-qing，传入产品定义 + 调研深度
  ↓
顾清：执行调研（识别竞品 → 多维分析 → 框架分析 → 输出报告）
  ↓
周明：审查报告，确认差异化机会清晰、可执行
  ↓
周明：召集评审会，让其他角色挑战
  ↓
周明：收敛决策点（选哪条差异化路径），等用户确认
```

## 关键原则

- 全程遵循诚实协议：关键结论标置信度、声明数据缺口、不回避竞品优势
- 「差异化机会」与「可落地建议」是承接下游阶段的关键，必须明确、可行动
- 获得用户批准后再进入阶段1
