## 一、产品定位与背景

### 1.1 产品名称
**ConvIntel**（度小满高净值客户对话智能分析平台）

### 1.2 一句话定义
为度小满金融打造一个面向内部员工的 **per-user AI 客服个性化平台**，通过对话自动分类、用户画像生成、个性化 prompt 闭环三件套，让每一位高净值借贷客户的下一次 AI 客服对话都基于其专属画像 + 专属 prompt 来响应 [依据 decision-00.5-001]。

### 1.3 业务背景
- **行业数据**：金融业高净值客户因 AI 客服模板化体验流失占比 **27%**（艾瑞 2025 白皮书）
- **度小满数据**：高净值借贷用户人均 LTV 是普通用户 **3.8 倍**（度小满 2024 财报）
- **同业现状**：招行、平安、微众均做了通用客服 AI，但**无金融机构实现 per-user prompt 个性化**——本产品填补行业空白
- **技术基建**：度小满已有「小满智客」LLM 平台，提供 `system_prompt` + `user_context` 字段动态注入接口，**本产品作为上层 prompt 工程平台**

### 1.4 目标用户与使用规模
| 用户角色 | 数量预估 | 使用频率 | 主要任务 |
|---|---|---|---|
| 人工客服 | 200-300 人 | 每日 8 小时高频 | 接入对话、查看画像、修正分类 |
| AI 产品同事 | 5-15 人 | 每日 2-4 小时 | 审核 Prompt、监控指标 |
| 客服质检/培训 | 10-30 人 | 每日 1-2 小时 | 按类别筛选、共性问题分析 |
| 高净值用户运营 | 5-10 人 | 每周数次 | 查看画像、跨用户分析 |

## 四、详细功能规格

### F1：对话列表页

**入口**：左导航「对话」  
**布局**：三栏（256/auto/360）

**核心字段（表格列）**：
| 列 | 数据类型 | 示例 |
|---|---|---|
| 客户脱敏名 | 字符串 | `王*** (139****8821)` |
| 高净值标识 | 徽章 | `[VIP]` 金色 |
| 主分类 | 4 类徽章 | 蓝/紫/绿/红 |
| 对话首句摘要 | 字符串 | "我借了 80 万，最近发现利率…" |
| 上次接入时间 | 时间 | `2026-06-15 14:32` |
| 已读未读 | 状态 | 未读小蓝点 |
| 客服 | 字符串 | "AI" / "人工·张玲" |
| 对话轮数 | 数字 | 12 |

**筛选器**：
- 用户搜索（脱敏名 / 手机后 4 位）
- 4 类徽章筛选（点亮切换）
- 时间范围（今日/本周/本月/自定义）
- 「仅高净值」开关
- 已读未读切换

**性能要求**：
- 首屏 100 条 < 1.5s
- 滚动至 500+ 条无卡顿

### F2：对话详情页

**入口**：列表行点击 / 详情页快捷键 J/K

**布局**：严格三栏

**左栏 256px**：精简列表（客户名 + 主分类徽章），当前项高亮，点击切换

**中区**：
- 顶部固定栏：客户脱敏名 + 高净值徽章 + 主分类徽章 + 「修正分类」按钮
- 主区消息气泡：
  - 用户：右橙气泡 #F97316 70%
  - AI 客服：左灰气泡 #21262D
  - 人工客服：左度小满蓝 70% 透明 [Stage 01 评审会修订]
  - 每条带时间戳和发送者标识
- 底部悬浮快捷操作栏：标记重要 / 触发重新分类 / 跳画像页 / 上一条 / 下一条

**右栏 360px**：标签页切换
- **画像速览（默认）**：
  - 顶部接待速览卡（黄底 + ⛔ 雷区行红描边 [Stage 01 评审会修订]）
  - 折叠的完整画像
- **分类详情**：
  - 主分类 + 置信度数字（70-99%）
  - AI 推断依据（高亮关键词）
  - 历史分类时间线
- **Prompt 历史**：该用户已上线/待审核 prompt 列表

**键盘快捷键**：
- `J/K`：上下条切换
- `L`：标记已读
- `R`：触发重新分类
- `P`：跳到 Prompt 中心
- `ESC`：返回列表

### F3：AI 自动分类

**目标**：≥ **80%** 准确率（[依据 decision-00.5-002]，从原 85% 调整为 80%）

**4 类定义**：
| 分类 | 关键词信号 | 颜色 | 占比预估 |
|---|---|---|---|
| 借贷额度 | 提额、降额、申请、批不下 | 蓝 #3B82F6 | 38% |
| 利率不满 | 利率、年化、息费、IRR | 紫 #8B5CF6 | 27% |
| 还款方式 | 提前还、分期、期数、违约金 | 绿 #10B981 | 19% |
| 投诉 | 投诉、举报、客服态度、不合理 | 红 #EF4444 | 16% |

**模型方案**：
- 初版：度小满内部 Llama 微调模型 + Few-shot prompt（境内大模型 [依据 CAC 第 7 条]）
- 数据：ConversationLake 标注 1 万条对话（5 个标注员 × 2 周）
- 部署：内部 GPU 服务器，QPS ≥ 100

**置信度阈值**：
- ≥ 80%：直接落库主分类
- 60-80%：标记低置信度，UI 上⚠角标，进 F6 人工修正队列
- < 60%：兜底为"借贷额度"（占比最高的类），进人工修正队列

### F4：用户画像页

**布局**：三栏（256/auto/360）

**中区核心区**：
1. **顶部基本信息卡**（高度 96px）：

2. **接待速览卡**（高度 132px，置顶 + 黄底 [Stage 00 评审会共识]）：

   - 第 2 行 ⛔ 雷区警示（细红描边 [Stage 01 评审会修订]）
   - 第 3 行 🌡️ 当前情绪（细黄描边 [评审会修订]）

3. **AI 总结卡**（灰底）：
   - 性格特征（带 AI 图标 + 置信度 0-100%）

## 五、用户流程

### 流程 1：人工客服日常处理（高频）
```
登录 SSO → 工作台 → 看待办 KPI → 进对话列表 →
按"未读+高净值"筛选 → 点击对话 → 看右侧速览卡 →
浏览中区对话 → 必要时跳画像页深读 →
处理完毕 → 标记已读 → 系统弹出 7 秒可关闭的"客服备注引导"（不强制）[评审会修订] →
回列表
```

### 流程 2：AI 产品审核 Prompt（每日）
```
登录 → 工作台「待审核」KPI 显示 7 → 点击进 Prompt 中心待审核 Tab →
左栏看用户画像 + 速览卡 → 中区读对话历史 →
右栏审 AI 生成 prompt → 通过/修订/驳回 →
回列表，循环
```

### 流程 3：客服主管按类筛选（每周）
```
登录 → 进对话列表 → 仅勾选"投诉"红徽章 →
时间范围"本周" → 看汇总数据 →
点开 5-10 条样本 → 写共性总结发邮件
```

### 流程 4：高净值客户全链路（系统视角）
```
高净值客户在度小满 APP 发起对话
  → 小满智客 LLM 调用本平台 fetch_user_prompt(user_id) API
  → 本平台返回该用户当前生效的 system_prompt + user_context
  → 小满智客带专属上下文回复用户
  → 对话结束后，conversations 表新增记录
  → F3 自动分类 Worker 拉取并打标
  → 累积到阈值（如 ≥3 通新对话）触发 F4+F5 重新生成画像和 Prompt
  → AI 产品审核员队列收到待办
  → 审核通过 → fetch_user_prompt 返回新版本
```

## 六、技术架构

### 6.1 系统总体
```
┌─────────────────────────────────────────────────┐
│  ConvIntel Web (前端 React/Vue)                   │
│  ├─ 7 个核心页面 + SSO Auth                       │

│  ConvIntel Backend (Node.js / Java)              │
│  ├─ API 网关 + 鉴权                                │
│  ├─ 对话/画像/Prompt 业务服务                       │
│  ├─ 分类 Worker（异步队列）                          │
│  └─ 数据回流 ETL                                   │
└─┬───────────┬──────────────┬──────────────────┘
  │           │              │
  ▼           ▼              ▼
[小满智客]  [Conv Lake]    [ConvIntel DB]
LLM 平台   原始对话湖      画像/Prompt/标注
```

### 6.2 关键接口

#### 与小满智客集成（**含降级与熔断 [评审会修订]**）
```
POST /xiaoman/v1/conversation/personalized
  SLA: 高峰期 1s 内必返
  降级策略：本平台 5xx / timeout 时，小满智客自动使用 fallback_prompt（通用 prompt）继续服务
  熔断：连续 10 次失败后熔断 30s，期间全部走 fallback
```

#### 部署架构 [评审会修订]
- 沿用度小满现有 K8s 集群，独立 namespace `convintel-prod`
- 异步队列采用度小满已有的 **RocketMQ** 集群（不新增中间件）
- 数据库 MySQL（画像/Prompt）+ Redis（缓存）+ ClickHouse（看板分析）

#### ConversationLake 拉取
```
GET /convlake/v1/conversations
  Params: user_id / time_range / hi_value=true
  Returns: [{conv_id, messages, ai_or_human, ...}]
```

### 6.3 LLM 选型决策（[依据 decision-001 风险登记]）
- 分类：度小满 Llama 微调（Stage 0.5 已锁定，CAC 合规）
- 画像生成：同上
- Prompt 生成：同上
- ❌ 禁用 OpenAI/Anthropic（CAC 第 7 条）

### 6.4 数据合规
| 合规项 | 实现 |
|---|---|
| 境内大模型（CAC §7） | 全链路使用度小满 Llama |
| 可解释/申诉/删除（CAC §11） | F4 申诉与删除按钮，操作留痕 |
| 人工干预（CAC §15） | F5 单人审核必经 [依据 decision-001] |

| 操作审计 | 所有审核/修正/删除操作 immutable log |
