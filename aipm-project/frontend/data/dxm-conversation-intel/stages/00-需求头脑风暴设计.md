## 一、产品定义

- **一句话定义**：面向度小满内部员工的高净值借贷客户对话智能分析平台，通过自动分类、画像生成、个性化 AI Prompt 三件套，把"标准化 AI 客服"升级为"千人千面的高质量 AI 客服"。
- **目标用户**：
  - 主要：人工客服团队（高频日常使用）+ AI 产品团队（生成与审核 prompt）
  - 次要：客服质检团队、用户运营团队
- **核心问题**：度小满 APP 内的 AI 客服回复过于标准化，对高净值借贷用户的精细化诉求理解不到位，导致服务体验差、流失风险高。
- **产品形态**：B2B 内部工具（Web 后台），支持 SSO 单点登录到度小满内网。
- **首版目标**：让度小满每一位高净值借贷客户的下一次 AI 客服对话，都基于其历史对话画像生成的个性化 prompt 来响应。

## 二、用户场景与任务流

### 场景 1：人工客服接入对话（高频）
```
高净值用户在度小满APP发起对话
  → 系统识别为高净值
  → 客服后台弹出"用户画像速览卡"（基础信息 + AI总结 + 历史分类分布）
  → 客服基于画像有针对性应对
```

### 场景 2：对话结束后自动分类（实时）
```
用户对话结束（无论 AI 还是人工）
  → 后端 NLP Agent 自动分类（4 类：额度/利率/还款/投诉）
  → 推送到对话列表页
  → 高置信度直接落库；低置信度高亮等人工修正
```

### 场景 3：客服主管按类筛选（每日/每周）
```
主管打开对话列表
  → 按"投诉"筛选今天所有对话
  → 发现共性问题（如多人投诉某活动）
  → 上报或触发批量处置
```

### 场景 4：AI 产品团队 prompt 闭环（核心目的）
```
某高净值用户对话累积到阈值（如 ≥3 通）
  → 系统检测并触发"画像生成 + Prompt 生成"
  → AI 产品同事在审核中心收到待办
  → 单人审核通过 → 个性化 prompt 上线
  → 该用户下次进入 APP 客服会话时，AI 用专属 prompt 回复
```

## 三、功能范围（P0 / P1 / P2）

### P0（首版必做，8 个功能）

| 编号 | 功能 | 说明 |
|---|---|---|
| F1 | 对话列表页 | 按用户/分类/时间筛选，三栏布局，支持高密度浏览 |
| F2 | 对话详情页 | 三栏：左列表 / 中对话原文 / 右画像分类操作 |
| F3 | AI 自动分类 | 4 类（借贷额度/利率不满/还款方式/投诉），彩色徽章 |
| F4 | 用户画像生成页 | 基础信息 + AI 总结（性格/诉求/沟通偏好/风险/服务建议） |
| F5 | 个性化 AI Prompt 生成与审核 | AI 触发生成 → 单人审核 → 上线生效 |
| F6 | 分类标签人工修正 | 员工可修改 AI 分类结果，回流训练 |
| F8 | 数据看板 | 分类分布、热点问题、AI 准确率、Prompt 通过率 |
| F9 | Prompt 版本管理 | 与 F5 合并为"Prompt 中心"，对比/回滚 |

### P1（二期）

| 编号 | 功能 | 说明 |
|---|---|---|
| F7 | 投诉自动告警/工单 | 检测到投诉自动生成工单 |
| F10 | 多客服协同备注 | 同一用户的服务建议多人共编 |

### P2（暂不做）
- 跨用户同类对话聚类分析
- 客服个人 KPI 看板
- 用户情绪曲线可视化

## 四、页面与信息架构

```
登录页 (SSO)
  └─ 工作台首页（看板 + 待处理 prompt）
      ├─ 对话列表页（核心入口）★主路径
      │   └─ 对话详情页（三栏）
      │       ├─ 跳→ 用户画像页
      │       └─ 跳→ Prompt 中心
      ├─ 用户画像详情页
      │   ├─ 关联对话
      │   └─ 关联 prompt
      ├─ Prompt 中心（审核 + 版本管理合并）
      │   ├─ 待审核
      │   ├─ 已上线
      │   └─ 历史版本对比
      └─ 数据分析看板
```

**核心导航路径（路径 1：对话驱动型）**：
```
工作台 → 对话列表 → 对话详情 → 点用户名 → 用户画像 → 看Prompt历史
```

## 五、交互与视觉

### 5.1 视觉风格

**数据密集型工具风格**（参考 Bloomberg Terminal / 量化交易终端）：
- 深色背景（#0E1117 主底，#1A1F29 卡片底）
- 高对比文字（#E6EDF3 主文 / #8B949E 次文）
- 品牌色：度小满蓝（#2967FF）
- 分类徽章：
  - 借贷额度 → 蓝色 #3B82F6
  - 利率不满 → 橙色 #F97316
  - 还款方式 → 绿色 #10B981
  - 投诉 → 红色 #EF4444

### 5.2 信息密度

**高密度**：员工每天处理几十上百条对话，每屏看 30+ 条对话项，类似 Excel/邮件客户端列表风格。

### 5.3 核心组件

- **三栏布局**：256px 左导航 + 自适应中区 + 360px 右侧栏
- **彩色分类徽章**：色块 + 一级分类文字
- **画像卡片**：双区分块——上半部 AI 灰底 + 下半部"客服备注" 浅黄底，视觉清晰区分

### 5.4 交互细节

- 对话详情页支持快捷键：`J/K` 切换上下条、`L` 标记已读、`R` 触发重新分类
- 画像页"客服备注"实时保存，3 秒后右下角 Toast 确认
- Prompt 审核支持"通过 + 备注"或"驳回 + 修改建议"

## 七、待评审会挑战的风险点

### ⚠️ 风险 1：数据合规（用户重点关注 B）
- 高净值用户对话涉及金融隐私（额度、利率、还款等敏感信息）
- 对话原文是否能传到外部 LLM？是否必须用度小满内部模型？
- AI 生成的画像是否构成"对客户的画像歧视"？是否有合规边界？
- 个性化 prompt 是否需要监管报备？

### ⚠️ 风险 2：AI 准确性（用户重点关注 C）
- 4 类分类的边界并不天然清晰（"利率不满"是否常常和"投诉"重叠？）
- 画像总结的"性格特征"是 AI 推断，可能给员工误导
- 个性化 prompt 上线后如果效果不好，影响用户体验，怎么回滚？
- "高置信度"和"低置信度"的阈值怎么定？

### ⚠️ 风险 3：用户体验（用户重点关注 D）
- 客服每天接入对话很急，画像速览卡是否会成为"再多看一秒就慢了"的负担？
- AI 产品团队审核 Prompt 工作量是否可承受？（如果 100 个高净值用户，每天产生几十个待审核）
- 三栏 + 高密度的视觉是否真的不疲劳？

### ⚠️ 风险 4：北极星指标的可达成性
- "Prompt 通过率 ≥ 70%"在第一版就达成是否乐观？
- "用户满意度 +10%"如何归因到这个产品上（不被其他客服优化项干扰）？

### ⚠️ 风险 5：MVP 范围
- 5-6 周做完 8 个功能 + AI 模型，工程量是否被低估？
- 是否应该 F4（画像）和 F5（Prompt）拆到二期？
