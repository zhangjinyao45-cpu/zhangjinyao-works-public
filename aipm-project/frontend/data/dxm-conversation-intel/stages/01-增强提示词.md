## 一、产品一句话提示词（用于 Stage 02-04 共享上下文）

> 「**为度小满金融打造一个面向内部员工的 per-user AI 客服个性化平台——通过对话自动分类、用户画像生成、个性化 prompt 闭环三件套，让每一位高净值借贷客户的下一次 AI 客服对话都基于其专属画像 + 专属 prompt 来响应；技术上深度集成度小满已有「小满智客」LLM 平台，通过 system_prompt + user_context 字段注入；合规上保留 100% 人工审核与可申诉机制；视觉上采用深色交易终端风（点缀度小满蓝），三栏数据密集型布局**」

## 四、核心页面提示词（每页一段）

### 页面 1：登录页 (`login.html`)
> 极简单页登录卡片，居中。卡片带度小满 logo + "ConvIntel · 高净值客户对话智能分析平台"标题。仅一个"使用度小满 SSO 登录"按钮（#2967FF 实色），点击 mock 跳转。背景大面积深色 #0E1117，居中卡片 #1A1F29。底部脚注"度小满风险与合规·内部使用"。

### 页面 2：工作台首页 (`dashboard.html`)
> 三栏布局：
> - **左 256px 导航**：Logo + 6 个一级菜单（工作台/对话/用户画像/Prompt 中心/数据看板/设置），当前项 #2967FF 左竖条 + 浅蓝背景
> - **中区主内容**：
>   - 顶部 4 张 KPI 卡片：今日新对话数 / 待审核 prompt 数 / 分类准确率 / 高净值用户数（每卡 24px 数字 + 12px 文案 + 趋势小图）
>   - KPI 下方：左侧 60% 是「我的待办」（待审核 prompt 列表，每项一行：用户名脱敏 + 当前类别徽章 + 生成时间 + "审核"按钮），右侧 40% 是「今日分类分布」环形图 + 4 个类别图例
> - **右 360px 侧栏**：「最近活动」时间轴（员工自己的最近 10 条操作）

### 页面 3：对话列表页 (`conversations.html`)
> 三栏：
> - **左导航**：同上
> - **中区**：
>   - 顶部筛选条：用户搜索框 + 4 个分类徽章筛选（点亮/灰显切换） + 时间范围 + "仅高净值"开关
>   - 主表格：高密度行（每行 36px）—— 列依次：客户脱敏名 / 高净值标 / 分类徽章（紫=利率/蓝=额度/绿=还款/红=投诉）/ 对话首句摘要 / 上次接入时间 / 已读未读 / "查看"
>   - 列表至少 20 行起，分页 50/100/200 可选
> - **右侧栏**：当前选中对话的「快速浏览卡」（脱敏画像速览 + "进入详情"按钮）

### 页面 4：对话详情页 (`conversation-detail.html`)
> 严格三栏：
> - **左 256px 列表**：同列表页缩略版（仅显示客户名 + 分类徽章），点击切换
> - **中区对话原文**：消息气泡风格（用户右橙气泡 / AI 左蓝气泡 / 人工客服左绿气泡），每条消息带时间戳和发送者标识。顶部固定栏：客户脱敏名 + 高净值徽章 + 当前 4 类单标签徽章 + "修正分类"按钮。底部悬浮"快捷操作栏"（标记重要 / 触发重新分类 / 跳画像页）
> - **右 360px**：标签页切换「画像速览 / 分类详情 / Prompt 历史」
>   - 画像速览：3 行接待速览卡（黄色高亮：开场建议 / 雷区警示 / 用户当前情绪），下方折叠的完整画像
>   - 分类详情：当前主分类 + 置信度数字 + AI 推断依据
>   - Prompt 历史：该用户已上线 prompt 列表

### 页面 5：用户画像页 (`user-profile.html`)
> 三栏：
> - **左 256px**：用户搜索 + 高净值列表
> - **中区**（核心）：
>   - 顶部基本信息卡：脱敏姓名 + 手机后 4 位 + 高净值标 + 借贷额度 / 利率 / 还款情况 / 累计对话数 / 各分类数量分布
>   - 下方双区拼接：
>     - 上半部「AI 总结」灰底卡：性格特征 / 核心诉求 / 沟通偏好 / 风险标签 / 服务建议（每项带 AI 图标 + 置信度标识）
>     - **接待速览卡**（顶部置顶 + 黄底色）：3 行 — 开场怎么说 / 不能说什么 / 当前情绪状态
>     - 下半部「客服备注」浅黄底卡：富文本编辑区，下方"实时保存中..."提示，3 秒后变绿 ✓
>     - 「申诉/删除」操作区（合规要求）：用户可申请删除画像 / 修正画像（[依据 CAC 第 11 条]）
> - **右 360px**：该用户的「关联对话时间轴」+「专属 Prompt 状态」（V1.2 已上线 / V1.3 待审核）

### 页面 6：Prompt 中心页 (`prompt-center.html`)
> 顶部三标签页：「待审核 / 已上线 / 历史版本」
> 
> **待审核 Tab**（核心）—— 严格三栏（[依据 Stage 00 苏予共识]）：
> - **左 320px 用户画像**（紧凑版）：基本信息 + 接待速览卡 + AI 总结摘要
> - **中区 对话历史**：该用户最近 5 通对话，分类徽章左侧标记，可滚动
> - **右 360px Prompt 编辑区**：
>   - 上半 AI 生成的 system_prompt 文本（带语法高亮的 prompt 模板）
>   - 中间 user_context 字段
>   - 下半「通过」（#2967FF 大按钮 + 备注输入框）/「驳回 + 修改建议」（红色边框按钮）
>   - 工作流：审核员可直接编辑修订 → 点"上线"
> 
> **已上线 Tab**：表格视图，列：用户 / Prompt 版本号 / 上线时间 / 命中对话数 / CSAT 表现 / "回滚"按钮
> 
> **历史版本 Tab**：版本对比视图（左右双 diff），可回滚到任意旧版本

### 页面 7：数据分析看板页 (`analytics.html`)
> 顶部时间筛选 + 维度切换。主区四象限：
> - 左上：分类分布趋势图（4 条折线，色板对应）
> - 右上：分类准确率仪表盘（目标 80% 横线 [依据 decision-00.5-002 调整后]）
> - 左下：Prompt 通过率柱状图（目标 70%）
> - 右下：Top 10 高频问题词云

## 六、状态与边缘场景提示词（Stage 04 必须实现）

| 状态 | 触发场景 | 视觉表达 |
|---|---|---|
| Loading | 拉取对话列表 | 骨架屏占位（与最终内容形态一致），不要 spinner |
| Empty | 当前筛选无结果 | 居中插图 + "暂无符合条件对话" + "重置筛选"按钮 |
| Error | API 失败 | 顶部红条 + 错误描述 + "重试"按钮 |
| Disabled | Prompt 上线后不可再次审核 | 按钮 50% 透明 + 鼠标禁用样式 |
| Success | Prompt 审核通过 | 右下 Toast 绿色 + 自动消失 3s |
| ConfidenceLow | 分类置信度<70% | 徽章右上挂"⚠"图标，悬浮显示置信度 |
| HighNetWorth | 高净值用户对话 | 整行左侧 3px 金色竖条 + [VIP] 徽章 |

## 七、Stage 04 原型实现强约束

1. **单一 HTML 文件**，所有 CSS 写在 `<style>` 内、JS 写在 `<script>` 内
2. **CDN 库允许**：Tailwind / Chart.js / Lucide Icons / Day.js
3. **页面切换用 SPA 风格**：URL hash 路由（#dashboard / #conversations 等）
4. **真实 mock 数据**：至少 30 条对话、6 个用户画像、5 个待审核 Prompt
5. **可点击交互**：左导航点击切页 / 列表点击进详情 / 审核按钮点击弹 Toast
6. **响应式不强制**：仅保证 1280px 以上正常
7. **每个页面都要有真实数据感**，不是占位文字
8. **mock 数据中至少包含 2 个真正的"per-user prompt"完整文本**，体现产品差异化
9. **接待速览卡必须出现在画像页和对话详情页**
