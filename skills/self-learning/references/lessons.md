# 经验与拒绝记录

> 本文件由 `self-learning` 在每次执行时追加。

## 格式

```
### YYYY-MM-DD HH:MM
- **来源**：<source（已脱敏）>
- **新增**：N 条
- **丢弃**：M 条
  - "<被丢弃的描述（已脱敏）>" → 原因：<边界命中 / 重复 / 不可复用 / 脱敏复检命中>
- **冲突**：K 条
  - "<旧>" vs "<新>" → 处理：keep_b（保留新版，更具体）
- **脱敏命中**：P 条（按类别）
  - 例：`PHONE: 2, TOKEN: 1, EMAIL: 3`
  - **禁止**在此处记录原值或可还原片段，仅记类别与条数。
- **tmp 清理**：是 / 否（若阶段 1 走过 tmp 缓冲，必须显式标注是否已删除）
- **写入前复检**：通过 / 拒绝 X 条（拒绝则给出类别）
- **身份更新（阶段 7）**：跳过 / 已应用 / 因脱敏拒绝 / 因核心字段越权拦截
  - 触发条件：user_explicit / stable_preference / stable_context
  - 变更字段（不含原值）：例如 `recent_focus, preferred_tools`
  - run id：与 `identity-changelog.md` 中条目一一对应
```
