# PRD-HTML-ZJY 同步技能

基于PRD变更同步更新HTML原型，保留历史版本支持回滚。

## 使用方式

### 同步更新
**触发词**: "同步原型"、"更新HTML"、"PRD改了"

**执行**:
1. 读取 `02-产品需求文档.md` 和 `04-UI交互原型.html`
2. 分析PRD变更内容
3. 基于PRD重新生成/更新HTML原型
4. 保存旧版本到历史目录
5. 输出生成摘要

### 查看历史
**触发词**: "查看原型历史"

**输出**: 显示最近5个版本的时间戳和变更摘要

### 回滚版本
**触发词**: "回滚到v{N}"、"恢复版本{N}"

**执行**: 将指定版本恢复为当前HTML

## 文件结构

```
workspace/
├── 02-产品需求文档.md          # PRD源文件
├── 04-UI交互原型.html          # 当前HTML原型
├── .prd-html-sync/             # 同步数据目录
│   └── history/                # 历史版本（最多5个）
│       ├── v1_20240402_154500.html
│       ├── v2_20240402_161200.html
│       └── ...
└── sync-report.md              # 最新同步报告
```

## 同步流程

```
1. 读取PRD + HTML
2. 备份当前HTML到 history/v{N+1}_{timestamp}.html
3. 如果历史版本>5，删除最旧的
4. 基于PRD生成新HTML（保留原有样式风格）
5. 写入 04-UI交互原型.html
6. 生成 sync-report.md
7. 输出: ✅ 同步完成，当前版本v{N+1}
```

## 版本管理规则

- **保留数量**: 最多5个历史版本
- **命名格式**: `v{版本号}_{YYYYMMDD}_{HHMMSS}.html`
- **清理策略**: 超出5个时删除最旧的
- **回滚**: 复制指定版本覆盖当前HTML

## 输出示例

**同步完成**:
```
✅ 同步完成

版本: v3
变更: 基于最新PRD重新生成
历史: 已保留v1, v2（共3个版本）
报告: sync-report.md
```

**查看历史**:
```
历史版本:

v3 (当前) - 2024-04-02 18:00
v2 - 2024-04-02 16:12
v1 - 2024-04-02 15:45
```

**回滚完成**:
```
✅ 已回滚到v2
当前HTML已恢复为2024-04-02 16:12版本
```

## 边界处理

- **首次同步**: 直接生成，无历史版本
- **PRD未变**: 提示"PRD无变更，跳过同步"
- **历史满5个**: 删除最旧的，保留最新的5个
- **回滚不存在版本**: 提示"版本v{N}不存在"
